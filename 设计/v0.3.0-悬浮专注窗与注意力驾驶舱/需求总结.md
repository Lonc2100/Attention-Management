# 悬浮专注窗与注意力驾驶舱 — 需求总结

> 版本：v0.3.0
> 日期：2026-07-14
> 状态：PRD 与版本边界已确认，主视觉稿已同步最新时间轴决策

## 背景

当前“今日概览”把计划提醒、当前上下文、多个同等大小的指标、成果空状态、项目排行、应用时间和采集控制堆叠在同一页。页面虽然已经可用，但首屏被大标题与大卡片占据，真正重要的时间比例和全天节奏缺少清晰表达。

## 目标

1. 让用户在首屏看清今天的电脑时间去向和发生时段。
2. 用统一分母的圆环同时表达 Codex 项目和其他应用，并明确 Codex 父子关系。
3. 将计划、复盘、数据、设置和诊断严格分工。
4. 用悬浮窗持续显示当前项目和连续专注时间。
5. 为未来跨应用项目识别预留可靠的置信度表达。

## 功能范围

### 做什么

- 重构今日概览为注意力数据驾驶舱。
- 增加统一分母的注意力圆环。
- 增加应用事实层与项目判断层并列的全天时间轴。
- 压缩重复指标、计划提示和当前上下文展示。
- 设计置顶可折叠的悬浮专注窗。
- 表达已确认、推断中、未确认和 AFK 状态。

### 不做什么

- 本阶段不修改正式应用代码。
- 不把低置信度跨应用时间直接计入项目。
- 不在今日概览编辑计划、填写复盘或控制采集。
- 不直接复制 ActivityWatch 的 Vue 界面代码，只借用数据能力和成熟的时间轴表达。
- 不在这一阶段打包或发布 v0.3.0。

## 载体

已有 Windows Electron 应用中的“今日概览”页面，以及一个独立的桌面悬浮窗。悬浮窗默认置顶并可折叠，用户可以在设置中切换为桌面层。

## 关键决策

| # | 决策 | 原因 |
|---|------|------|
| 1 | 采用 A 方案：圆环与时间轴并列 | 同时回答“时间去了哪里”和“什么时候发生” |
| 2 | 圆环分母为电脑有效活跃时间 | 项目和其他应用可以直接比较 |
| 3 | Codex 父级由项目叶子节点替代 | 避免父子时间重复计算 |
| 4 | 首页只承担数据驾驶舱职责 | 计划、复盘和系统控制不再挤压核心数据 |
| 5 | 时间轴保留应用事实层和项目判断层 | 推断错误不会污染原始记录 |
| 6 | 跨应用连续专注继续，但项目归属分级表达 | 兼顾连续性与准确性 |
| 7 | 悬浮窗默认置顶可折叠 | 保证可见性，同时控制打扰程度 |
| 8 | 视觉继续使用深色与薄荷绿，但减少卡片和描边 | 保持产品一致性并建立清晰主次 |
| 9 | 时间轴移除独立 24 小时顶部指针，采用“活动时段自动放大 + 单轨时间块” | 避免重复指针与大片黑色空白，让用户直接读清项目边界和准确起止时间 |
| 10 | Codex 时间轴直接拆成具体项目，同属绿色家族但使用不同色阶与统一 `C` 标识 | 同时表达“都是 Codex”和“属于不同项目”，避免整条时间轴出现过多完全相同的绿色 |

## 技术约束

- ActivityWatch 提供前台应用、窗口标题和 AFK 原始事件。
- 当前项目拆分只对 Codex 前台注意力有可靠依据。
- 跨应用项目判断必须新增推断数据与置信度，不能只靠最后一次 Codex 上下文。
- 主设计稿必须离线打开，不依赖外部字体或图表 CDN。
- ActivityWatch 采用 MPL 2.0；借用数据 API 与交互思路比直接复制前端源码更适合当前 React 应用。
