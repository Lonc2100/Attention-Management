# v0.7.5 · Codex 可见上下文与 AFK 默认校准

状态：**代码与自动化验证完成；当前 Codex Desktop 真实侧边栏验证未通过，尚未作为已解决功能发布。**

## 改动

- 首次按 Harness-lite 分层：`providers/codex-visible-context` 只读取 Windows 可访问性信息；`services/codex-attribution-service` 只在可见聊天与 app-server 线程唯一匹配时保存归属；旧 `CodexContextTracker` 保留为计时兼容入口。
- Provider 新增已选侧边栏项的只读探测路径（`SelectionItemPattern`），并保留顶部项目条路径。
- 侧边栏只给出聊天标题时，重复标题一律拒绝归属；不会用最近活跃线程猜测。
- 新安装默认的主要离开阈值改为 5 分钟；已有有效值（包括 15 分钟）保持不变。
- 修复 AFK 段人工计入在窗口 watcher 恰好缺失时会变成不可撤销记录的问题：现在会显示为不伪装成应用或项目的“人工计入工作”条目。
- 首页、设置、E2E 与 README 同步为 5 分钟和真实的可用性边界。

## 真实验证证据

- 2026-07-21 在当前 Windows Codex Desktop（`OpenAI.Codex 26.715.8383.0`）运行 `RUN_CODEX_WINDOW_INTEGRATION=1`。
- Windows UI Automation 只返回 `RootWebArea`，没有暴露顶部条、侧边栏、列表项或当前聊天标题；Provider 返回 `null`，集成测试按预期失败在“必须有可见上下文”的验证闸门。
- 本机 Codex app-server schema 未发现“当前桌面选中聊天”接口；`thread/list` / `thread/loaded/list` 不能证明窗口当前选择，因此未接入为猜测回退。
- 这不是“功能已修复”的证据。当前环境中 Codex 项目时间仍会安全进入待分类，直到 Codex 提供可靠当前会话信号或 Windows 无障碍树可读。

## AFK 真实数据观察

- 当前已安装用户设置仍为 15 分钟，验证了升级不会静默覆盖既有选择。
- ActivityWatch AFK watcher 正常提供原始状态；采样时一段 AFK 为 212.4 秒，低于 5 分钟，因此按新策略也不会被误判为主要离开。
- 需要一次连续超过 5 分钟、随后恢复输入的真实人机回合，才能完成新阈值的正向现场验证；本版本不伪造该结果。
