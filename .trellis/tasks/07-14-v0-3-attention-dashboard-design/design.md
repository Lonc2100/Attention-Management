# Design — v0.3.0 悬浮专注窗与注意力驾驶舱

## Information Architecture

### 今日概览

1. 顶部：日期、采集状态、当前/最近确认项目。
2. 轻量计划摘要：只显示绝对优先成果和前往计划页的入口。
3. 主数据区：注意力圆环 + 自动聚焦实际活动范围的单轨时间轴。
4. 紧凑统计条：活跃、最长连续专注、AFK、项目覆盖率。
5. 明细区：注意力去向 + 应用明细。

### 页面职责边界

- 早间计划：编辑 1–3 个成果和唯一优先项。
- 晚间复盘：成果状态、主观效率、AFK 补记、AI 分析、明日方向。
- 设置：采集控制、开机启动、悬浮窗模式、提醒和隐私。
- 诊断：ActivityWatch、Codex CLI、上下文识别和自启动链路状态。
- 活动明细：v0.4.0 实现后加入侧栏；v0.3.0 不展示不可用占位入口。

## Data Semantics

### 圆环

圆环只展示互斥的叶子节点：

`非 Codex 应用 + Codex 已分类项目 + Codex 待分类 = 电脑有效活跃时间`

父级 Codex 仅作为分组标注，不再作为独立切片。示例使用当前截图的真实量级：电脑活跃 197 分钟，Codex 157 分钟，其中已分类 78 分钟、待分类 79 分钟，覆盖率约 49%。

### 时间轴

- 不保留独立的 24 小时顶部指针，避免重复信息和视觉噪音。
- 主时间轴自动放大到当天真实活动范围，并向前后各留约 30 分钟；示例范围为 08:00–13:00。
- 应用事实和项目判断合并在单个时间块内：项目/活动名称为主，应用与归属状态为辅，不再使用难以理解的上下双轨。
- 每小时显示明确刻度；较宽时间块直接显示起止时间，较窄时间块通过点击或悬停在下方详情栏显示准确起止。
- Codex 不作为单一大块，直接拆成具体项目；各项目使用同一绿色家族的不同明度和饱和度，并统一使用 `C` 标识或顶部色条。
- 已确认使用实心色；推断使用项目色斜纹；待分类使用低饱和琥珀；AFK 使用低对比点阵。
- 项目推断不能删除或改写应用事实数据，单轨只是显示合并，不改变底层数据分层。

## Visual Direction

- 风格：冷静、克制、本地优先的个人工作系统。
- 背景：近黑蓝灰；层级通过明度与间距建立，而不是大量边框。
- 主强调色：薄荷绿，仅用于当前状态、已确认项目和关键交互。
- 辅助色：蓝紫用于非 Codex 应用，琥珀用于待分类，灰色用于未知与 AFK。
- 卡片：只包裹真正独立的分析模块；指标使用紧凑文字，不再各占一张大卡片。

## Floating Widget

- 默认折叠胶囊：项目名、连续专注、确认状态。
- 点击展开：当前应用、项目归属、今日项目时间、打开主界面与暂时隐藏入口。
- 首次显示在主屏右上，支持拖动到任意屏幕边缘，并记住位置、显示器和折叠状态；显示器失效时回退主屏右上。
- 置顶是默认值；设置中可以改为桌面层或临时隐藏。
- 离开 Codex 后，连续电脑专注可以继续，但项目状态必须改为“推断中”或“最近确认”。
- 悬浮窗不提供手动切换项目；v0.3.0 不实现全屏应用智能避让。
