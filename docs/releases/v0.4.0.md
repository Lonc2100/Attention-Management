# v0.4.0 — 活动明细、人工纠错与归类规则学习

v0.4.0 把时间效率助手从“看聚合结果”推进到“检查证据、纠正错误、逐步学会”。

## 本次更新

- 侧栏新增独立“活动明细”，计划与复盘不再混入数据调查页面。
- 支持按日期查看真实记录区间、活动时间线、应用、窗口标题、项目和判断来源。
- 时间线与列表双向联动；选中后在右侧证据栏查看准确起止和完整标题。
- 支持把单段时间纠正到已有或新建项目，纠错可随时撤销。
- 可把纠错保存为未来归类规则：应用精确匹配 + 标题包含/完全匹配。
- 规则支持显式优先级、上移/下移、启用/停用和删除；第一条命中生效。
- 人工纠错 > 规则 > Codex 已确认上下文 > 应用/待分类，且所有页面共享同一分类引擎。
- 新规则只影响创建之后的活动，不会无提示重算历史；ActivityWatch 原始事件始终不改写。
- 本地存储升级到 v4，保留既有设置、计划复盘、Codex 样本和项目别名。

## 验证

- 30 个单元测试通过，另有 2 个依赖真实环境的集成测试按默认配置跳过。
- TypeScript 类型检查和生产构建通过。
- 12 条 Electron 端到端链路通过，其中活动明细覆盖新建项目、人工纠错、创建/删除规则、撤销纠错和横向溢出检查。
- 打包态、安装态和 Windows 自启动验证通过；真实用户数据从 v3 升到 v4 后，原有记录与 Codex 样本数量保持不变。

## 设计取舍

- 本版不开放正则表达式，避免日常纠错变成技术配置工作。
- 本版不默认回溯历史，避免规则变化造成过去报告静默漂移。
- 本版不做项目合并、批量重算或 AI 自动猜测；先通过真实使用反馈观察规则准确率。
