# v0.5.0 — 成果关联与个人规律

v0.5.0 把“记录了哪些时间”推进为“重要成果获得了哪些可信注意力”。成果与项目由用户在早间计划中显式关联，系统不会猜测关系，也不会要求用户在工作中手动切换计时器。

## 变化

- 早间每个成果可关联零到多个已知项目，关联与成果状态在重启后保留。
- 首页增加绝对优先成果证据，只累计显式关联项目的已确认注意力。
- 晚间复盘逐项显示关联项目与支持时间；应用、AFK、待分类时间不会被猜入成果。
- 新增“个人规律”页，可切换近 7/14/30 天，查看逐日活跃、优先成果注意力、结果、评分与上下文切换。
- 只有至少 3 个“优先成果完成、评分不低于 4 且存在关联项目注意力证据”的日期才会生成高质量候选时段，并始终标注相关性不等于因果。
- Codex CLI 复盘载荷加入成果对应的项目显示名和聚合分钟，不发送项目键、路径、对话标题或窗口标题流水。
- 本地 JSON 数据从 v4 迁移到 v5；旧成果自动获得空项目关联，其他记录与分类数据不变。
- 修正原 `npm run typecheck` 未真正检查引用项目的问题，主进程和渲染进程现在分别执行不落盘类型检查。
