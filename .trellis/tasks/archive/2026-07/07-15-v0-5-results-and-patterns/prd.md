# v0.5.0 成果关联与个人规律

## Goal

让用户能回答“最重要成果是否得到足够的已确认注意力、是否完成、明天应调整什么”，而不是只看到应用与项目排行榜。

## Requirements

### 1. 成果关联

- 每个早间成果可关联零到多个已有项目；绝对优先成果仍保持唯一。
- 关联为空时允许保存，但必须明确说明无法量化项目支持时间。
- 多个成果可以共享项目；每个成果单独显示支持时间，禁止把共享时间跨成果相加成总投入。
- v4 历史成果自动迁移为 `projectKeys: []`，不得丢失标题、优先级、复盘或纠错数据。

### 2. 当日成果证据

- 首页显示绝对优先成果、关联项目和今天已确认的支持注意力。
- 晚间复盘逐项显示完成状态、关联项目和已确认支持时间。
- 只计算 `kind=project` 且项目键明确匹配的非 AFK 时间；待分类与应用时间不自动猜入成果。

### 3. 个人规律

- 新增独立“个人规律”页，默认读取最近 7 天，可切换 14/30 天。
- 展示每天电脑活跃、绝对优先成果支持时间、完成状态、主观评分和上下文切换数。
- 候选高质量时段只来自已复盘、绝对优先成果完成、主观评分较高且存在关联项目注意力证据的日期。
- 少于 3 个合格日期时显示样本不足；所有规律结论必须标明相关性而非因果。

### 4. AI 复盘

- Codex CLI 载荷增加成果—项目映射和当日支持分钟数，不包含项目键、路径、任务 ID、完整窗口标题或对话正文。
- AI 输出必须以成果状态优先，并引用提供的聚合证据；数据不足时明确说明。

## Non-goals

- 不自动猜测成果与项目的关系。
- 不为每次应用切换增加手动计时或成果切换。
- 不引入统一生产力分数、因果结论、云端分析、团队目标或绩效监控。
- 不在本版实现备份恢复、数据导出、自动更新或 macOS。

## Acceptance Criteria

- [x] `A` v4→v5 迁移保留全部旧字段，并为旧成果补空项目数组。
- [x] `A` 计划可保存多项目关联；重新启动后关系保持。
- [x] `A` 首页与复盘只累计匹配项目的已确认时间；待分类和应用时间不进入成果证据。
- [x] `A` 多成果共享项目时各自可显示证据，但不存在跨成果总和指标。
- [x] `A` 7/14/30 天规律计算使用真实历史 ActivityWatch + 本地复盘，并对少于 3 个合格日返回样本不足。
- [x] `A` AI 载荷只包含项目显示名和聚合分钟，不包含敏感身份字段。
- [x] `A+H` 个人规律页在空数据、部分数据、断连和完整数据下均有可理解状态。
- [x] `A` Unit、typecheck、build、连续两轮 E2E、packaged smoke、升级安装与自启动全部通过。
- [x] `A` 真实 v4 数据升级后记录、样本、别名、规则与纠错数量不减少。

## Rollback

代码可整体 revert；v5 新字段为向后兼容的附加字段。v4 代码会忽略 `projectKeys`，回滚时不得删除用户数据目录。

## Approval

用户已通过连续版本 Goal Contract 预先授权普通产品、技术和 UI 默认决策；本任务无需额外设计确认。
