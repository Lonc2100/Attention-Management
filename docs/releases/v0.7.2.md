# v0.7.2 · 采集可靠性热修复

发布日期：2026-07-17

## 修复

- ActivityWatch stdout 改为不建立管道，消除持续日志写满父进程管道后“进程仍在但 API 卡死”的故障路径。
- 全年工作活动查询拆为最多 31 日一批；桶创建前日期直接记为已知零值，历史完成日保存为可重建派生缓存。
- 应用窗口、悬浮窗和托盘不再等待 ActivityWatch 最长 20 秒启动。

## 诊断与恢复

- 健康状态要求窗口和 AFK 桶持续产生新事件，同时兼容 `last_updated` 与 `metadata.end` 两种 ActivityWatch 元数据。
- 诊断页新增系统盘余量和自动恢复状态；低于 10 GiB 警告，低于 2 GiB 禁止重启。
- 连续三次数据面异常后才允许恢复；只结束本应用启动的 ActivityWatch，外部实例只报告异常。
- 自动恢复采用 1 分钟、5 分钟、15 分钟退避，并在重启后等待真实新事件。

## 上游版本决定

- ActivityWatch v0.14.0b1 已在独立端口和独立数据库完成 server、窗口 watcher、AFK watcher 与 Query API 冒烟。
- 它仍是预发布版，且未完成本产品安装包与多日真实运行验证，因此 v0.7.2 继续随包提供 v0.13.2。

## 边界

- 不删除、迁移或改写 ActivityWatch 原始数据库和 watcher 队列。
- 本版仍只支持 Windows x64，安装包仍未代码签名。
