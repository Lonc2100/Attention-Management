# v0.6.0 Windows 对外测试准备

## Goal

让新 Windows 测试者能够理解采集边界、建立可恢复的本地数据副本、导出不含窗口标题的报告，并在未签名状态下明确知道更新风险，而不是把开发者自用路径伪装成可公开分发产品。

## Requirements

### 1. 首次启动与信任说明

- 新安装/未完成引导的用户进入简短引导，依次说明本地采集、窗口标题风险、AFK 含义、Codex 项目归属边界和随时暂停入口。
- 引导必须显示 ActivityWatch 连通性与采集开关；无法连接时不能以“已准备好”结束。
- 不录屏、不采集键盘正文、不上传原始活动数据的承诺必须与实际实现一致。

### 2. 隐私控制

- 设置页可新增排除规则：按应用名或窗口标题包含条件排除派生注意力数据。
- 排除只影响本应用的聚合、时间轴、成果证据、个人规律、导出和 AI 摘要；不改写 ActivityWatch 原始 bucket。
- 规则可启停、删除；无标题/无效规则不得保存。

### 3. 备份、恢复与导出

- 支持导出可恢复的应用数据备份，包含 schema/version 元数据与计划、复盘、设置、别名、规则、人工纠错、项目关联；不默认打包 ActivityWatch 原始事件。
- 支持导出一份聚合 CSV 日报，不包含窗口标题、线程 ID、cwd 或项目键。
- 恢复前必须验证备份格式；恢复时自动写入本地恢复点，保留现有文件，可撤回到恢复前数据。
- 原始 ActivityWatch 数据仍由其自身官方导出渠道管理，应用在 UI 说明该边界。

### 4. 诊断与更新安全

- 诊断页可导出脱敏诊断包：版本、平台、连接状态、存储 schema、规则数量与错误摘要；不得包含路径、标题、线程 ID、AI 文本或活动流水。
- 设置页显示当前版本、安装包签名状态和更新政策。未签名时明确“不自动更新、不应假装可信发布”。
- v0.6 不引入网络自动更新、GitHub token、证书或遥测。

## Non-goals

- 不开始云同步、团队/账户、支付、录屏或键盘正文采集。
- 不修改或备份 ActivityWatch 原始 bucket；用户可使用 ActivityWatch 官方导出。
- 不将无证书安装包宣称为公开发布就绪。
- 不把本机新 profile 自动化替代真实外部用户验证。

## Acceptance Criteria

- [x] `A` 新 profile 首次启动可完成引导；ActivityWatch 断连时真实表达阻塞。
- [x] `A` 隐私排除规则影响所有派生展示与 AI 聚合，原始 ActivityWatch 事件不被写改。
- [x] `A` 备份导出、格式拒绝、恢复点、恢复和回退由自动化验证；真实安装升级后原数据未减少。
- [x] `A` CSV/诊断导出不包含原始窗口标题、cwd、thread id 或 project key。
- [x] `A` 设置明确未签名/无自动更新状态；无凭据时不运行签名或发布调用。
- [x] `A` Unit、typecheck、连续两轮 E2E、packaged smoke、升级、自启动和数据保留通过。
- [ ] `H` 用全新 Windows profile 的安装流程完成引导、采集、计划、备份/恢复和复盘；真实外部用户验证作为 v1.0 门禁单独记录。此项不能由本机自动化替代。

## Stop Boundary

Windows 代码签名或真实公开发布需要证书、费用或账号；真实外部用户可理解性也不能由本机自动化替代。命中时完成可验证的本地 beta 准备，保留明确 Handoff，不虚构验收。
