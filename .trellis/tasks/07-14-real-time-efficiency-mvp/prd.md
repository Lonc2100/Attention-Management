# 真实时间效率 MVP

## Goal

交付一个可安装、可开机自启的 Windows 独立应用，真实记录用户在电脑上的前台应用/窗口与离开状态，并用早间计划、晚间复盘和 Codex CLI 分析形成“计划 → 采集 → 复盘 → 次日优化”的闭环。

## Requirements

- ActivityWatch 随应用提供并由应用管理启动；界面必须展示真实连接状态，禁止演示数据冒充采集结果。
- 采集 Windows 前台应用、窗口标题、时间段和 AFK 状态；不录屏、不采集键盘正文。
- 早间计划最多 3 个重要成果，其中 1 个为绝对优先项；数据在重启后保留。
- 晚间复盘控制在约 5 分钟，展示真实时间线、应用汇总、专注块和离开时段，允许用户补记离开电脑期间做了什么。
- 效率判断以“重要成果完成情况 + 活动证据 + 用户主观判断”为依据，不把 AFK 自动判为低效。
- 通过本机已登录的 Codex CLI 生成真实分析；只传递聚合后的窗口/应用统计和用户填写内容，不上传原始全量窗口流水。
- 登录 Windows 后默认自动启动，可在设置中关闭；关闭主窗口后驻留系统托盘。
- 运行中按固定时间提醒；关机期间不提醒，下次启动时对错过的早间计划或晚间复盘进行补提醒。
- 提供诊断页，能检查 ActivityWatch、采集桶、本地存储、Codex CLI 和自启动状态。
- 所有开发过程由 Trellis 任务管理；开发源文件位于 `work/`，最终安装包和使用说明位于 `outputs/`。

## Acceptance Criteria

- [x] 首次启动可拉起随应用提供的 ActivityWatch，`/api/0/info` 返回成功。
- [x] 实际切换 Windows 前台窗口后，应用可显示新增的真实窗口事件与应用时长。
- [ ] 用户离开电脑超过 ActivityWatch 阈值后显示 AFK 时段，并可填写线下活动说明。
- [x] 早间 1–3 个成果与绝对优先项可保存，重启应用后仍存在。
- [x] 晚间复盘可保存成果完成状态、主观效率、总结和明日意向。
- [x] Codex CLI 能基于当天聚合信息生成分析，失败时显示可诊断错误而非伪造答案。
- [x] 默认开启 Windows 登录自启，设置开关可以真实修改系统登录项。
- [x] 错过提醒后再次启动，应用显示补提醒；已完成当天阶段则不重复提醒。
- [x] Windows 安装包可以生成，并通过安装/启动/核心链路冒烟测试。
- [x] 用户可以在应用内暂停/恢复采集，并清楚看到当前真实状态。

## Notes

- V1 仅支持 Windows，不包含手机与现实场景自动采集。
- 不依赖云端后端；ActivityWatch 和业务数据都保存在本机。
- 原型仅作视觉参考，不计入 MVP 完成度。
- Portable 外壳在最终验收机上未能启动内部主程序，已从交付物移除；正式交付只保留通过真实安装验收的 NSIS 安装版。
