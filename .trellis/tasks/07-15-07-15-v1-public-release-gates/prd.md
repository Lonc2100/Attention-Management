# v1.0.0 Windows 正式发布门禁

## Goal

在不扩大隐私边界、不虚构发布能力的前提下，确认 Windows 产品是否具备正式公开发布条件。未满足任一硬门禁时，不创建 `v1.0.0` 标签、不宣称正式版。

## 已验证基线

- v0.6.0 已有本地隐私控制、备份恢复、脱敏导出、自动化 E2E、打包 smoke、真实 v0.5→v0.6 升级与自启动验证。
- `TimeEfficiency-0.6.0-Setup-x64.exe` 的 Authenticode 检测为 `NotSigned`；当前没有代码签名证书、时间戳服务凭据或发布账号授权。
- 本机新 profile 只能验证实现链路，不能证明外部用户理解采集/隐私/恢复说明。

## 硬门禁

1. 由产品所有者提供合法 Windows 代码签名证书（或已授权的受信任签名服务与所需账号），并允许用于构建流程。
2. 至少一名真实、独立的 Windows 测试者在无开发环境的机器上完成安装、引导、真实采集、计划、复盘、备份恢复与卸载/数据处理理解，并留下可复核的反馈和问题结论。
3. 每个严重问题必须有复现、修复、回归证据；最终发布包必须重新完成升级、数据保留、自启动、签名验证与安全下载渠道验证。

## Non-goals

- 不购买证书、不创建/使用个人或组织密钥、不接入自动更新或遥测。
- 不把本机自动化、新 profile 或“可构建”说成真实外部用户验证。
- 不启动 macOS 开发；需要真实 Apple Silicon Mac 的 macOS 验收属于另一个目标。

## Done When

- 上述硬门禁全部有可审计证据，并完成 v1.0.0 版本 PRD、测试、发布说明、安装验证、tag/push。

## Stop Condition Evidence

- 当前已命中：缺少代码签名证书/相关费用和账号授权；缺少真实外部 Windows 用户验证。
- 根据连续版本 Goal Contract，这不是可由默认工程选择替代的条件。因此 v1.0.0 在这里停止，交由 Handoff 恢复。

## Handoff

1. 提供可用且已授权的 Windows 签名方案，或明确决定继续保留未签名本地 beta。
2. 指定真实外部测试者与其同意的反馈渠道；不要收集不必要的原始活动数据。
3. 收到两类证据后，从本任务继续，先审查证书使用范围与测试反馈，再实施必要修复和发布验证。
