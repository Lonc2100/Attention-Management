# Attention Management（时间效率助手）

面向 Windows 的本地优先时间效率助手。它把 ActivityWatch 的真实电脑活动、每日早间计划、晚间复盘和 Codex CLI 分析连接成一个可运行的个人工作闭环。

当前版本为 `v0.4.2`，已完成 Windows 安装包与真实链路验收，不是纯界面原型。

## 已实现

- 记录前台应用、窗口标题和 AFK（离开电脑）状态
- 当 Codex 真正在前台且用户未 AFK 时，自动按当前打开的 Codex 项目聊天拆分注意力；切换聊天无需先发送消息
- 显示最近确认的 Codex 上下文、项目分钟数、分类覆盖率和保守的“待分类”时间
- 首页注意力圆环以排除 AFK 后的全部电脑活跃时间为统一分母；Codex 具体项目作为叶子切片，不与 Codex 总量重复计算
- 圆环、项目图例与时间轴支持联动悬浮：提示卡固定在当前目标旁，不追随鼠标，并直接显示项目、类型、时长、占比和时间轴起止
- 项目级时间轴自动聚焦当天实际记录区间，显示应用、Codex 项目、待分类与 AFK 的准确顺序和起止
- 独立“活动明细”页面按日期展示时间线、应用、窗口标题、项目归属与判断来源，不再与计划/复盘混排
- 点击任意时间块或活动行可检查原始证据，并把错误时间纠正到已有或新建项目；纠错可随时撤销
- 可把一次纠错保存为“应用 + 标题包含/完全等于”的未来归类规则；支持排序、启停和删除
- 人工纠错始终高于规则，规则高于 Codex 自动上下文；原始 ActivityWatch 事件不会被改写
- 独立悬浮专注窗默认置顶，实时显示当前项目与连续专注时间；支持折叠、拖动、记忆位置、隐藏和桌面模式，展开后显示项目占比摘要
- 可更正自动识别出的项目显示名；更名不承担手动切换作用
- 每日填写 1–3 个重要成果，并指定唯一绝对优先项
- 晚间确认成果、主观效率和线下活动补记
- 使用本机 Codex CLI 生成复盘建议，不强制提供 API Key
- 默认随 Windows 登录启动，可在设置中关闭
- 关闭主窗口后留在系统托盘继续工作
- 电脑关机期间不执行提醒，下次启动时补提醒

## 隐私边界

- 本地优先，不录屏，不采集键盘输入正文
- ActivityWatch 原始事件保存在本机，可能包含前台窗口标题
- AI 只接收项目/应用聚合分钟数、成果文字、成果状态、主观评分和主动补记，不发送任务 ID、本机路径、对话名或完整窗口标题流水
- AFK 只说明用户离开电脑，不会自动判定为低效率

## 怎么使用

1. 登录 Windows 后应用默认在后台启动；双击托盘中的“时间效率助手”可打开主界面，托盘菜单也可恢复悬浮专注窗。
2. 早上填写 1–3 个重要成果，并选出唯一绝对优先项。
3. 正常使用 Codex。点开不同的项目聊天后，助手会在下一次采样内自动确认新上下文，**不需要先发送消息，也不需要在助手里手动切换项目**。
4. “今日概览”会用统一圆环和项目化时间轴呈现全部前台注意力。首次采样前或识别失败的 Codex 区间会显示为“待分类”，不会被强行猜到某个项目。
5. 要检查或纠正时，进入“活动明细”，点击时间线或活动行；右侧会显示应用、完整窗口标题、当前项目和判断来源。
6. 在“归入项目”中选已有项目，或选择“新建项目”。只改这一段就直接保存；若同类活动以后都应归到这里，勾选“以后同类活动自动归类”，确认标题条件后保存。
7. 新规则只影响保存之后的活动。可在“活动明细 → 归类规则”里上移、下移、停用或删除；人工纠错可在证据栏中撤销。
8. 自动项目名称不够准确时点“更正名称”；这只改报告显示，不改变识别边界。
9. 晚上用约 5 分钟确认成果、补记线下活动，并按需调用本机 Codex CLI 生成复盘建议。
10. 悬浮窗默认置顶；在“设置”中可改为允许其他窗口覆盖的桌面模式，也可暂停采集或重新显示悬浮窗。

自动分类依赖本机已安装并登录的 Codex CLI，以及 Windows UI Automation 暴露的 Codex 顶部项目名和聊天标题。它不录屏、不读取键盘输入，也不保存聊天正文或完整界面树。标题映射不到唯一线程时，该区间会进入“待分类”，不会继续误算到上一个项目。后台 agent 运行时间、Codex 被其他窗口遮住的时间和 AFK 时间都不会算作你的项目注意力。

## 本地开发

环境：Windows x64、Node.js、npm。

```powershell
npm install
npm run setup:activitywatch
npm run dev
```

常用验证与构建命令：

```powershell
npm test
npm run typecheck
npm run test:e2e
npm run dist
```

`setup:activitywatch` 会准备 ActivityWatch v0.13.2 本地运行时。项目识别使用 Windows UI Automation 读取当前打开聊天，再用 Codex 官方 app-server 协议映射稳定线程身份；app-server 返回值通过运行时 Schema 校验。运行时、依赖、构建目录和安装包均被 `.gitignore` 排除，不提交到仓库。
