# v0.7.0 首页工作活动与安静悬浮胶囊

## Goal

把长期工作节奏直接融合进今日首页，让用户打开应用后同时看见当前专注、今天投入、过去一年每日电脑投入，以及本周/本月的成果和专注结构；同时进一步缩小默认置顶悬浮窗，减少挡路和视觉干扰。

## Confirmed facts

- 当前安装并运行的是 v0.6.2；仓库 HEAD 另有主题 Token、页面模块和编辑型组件规则三组已提交但未打包、未安装、未推送的视觉基础改动。
- ActivityWatch 是唯一电脑活动事实来源；`activeSeconds` 已排除 AFK，但不能证明所有电脑活跃都是工作。
- 首页已有今天的应用/项目分布、成果证据和时间轴；本版必须在首页融合长期活动，不新增页面。
- 用户要求悬浮胶囊更小、更安静，优先减少挡路，普通视觉选择由执行者采用保守默认。
- 产品设计基线继续是“冷静的秩序，流动的注意力”：暖黑、奶油白、绿色注意力语义、细分隔线、无装饰性阴影、信息密度优先。

## Requirements

### R1. 首页工作活动图

1. 首页新增 `{ WORK ACTIVITY }` 模块，置于紧凑摘要之后、今天注意力分布与时间轴之前。
2. 模块提供 `每日 / 每周 / 每月` 三种聚合视图：
   - 每日：最近 366 天，每格一天；
   - 每周：由同一每日事实按周聚合；
   - 每月：由同一每日事实按月聚合。
3. 默认事实指标是“实际电脑投入”，定义为 ActivityWatch 前台窗口与非 AFK 时间的交集。界面不得把它无条件表述为确定工作产出。
4. 颜色使用固定绝对时长区间，不按当前样本动态重排；同一颜色在不同月份必须具有相同含义。
5. 无活动、无采集与查询失败必须可区分；查询失败不得伪装为零。
6. 鼠标悬浮和键盘焦点显示日期范围、实际电脑投入、日均投入或活跃天数；每日格在存在成果记录时补充优先成果状态与主要项目证据。
7. 点击每日格进入活动明细并定位对应日期；不可用格不可点击。

### R2. 日/周/月指标

1. 模块在当前选择范围内显示：
   - 本期实际电脑投入；
   - 日均投入（只使用范围内日期，明确活跃天数）；
   - 成果注意力占比（当天全部已关联成果项目的并集，重复关联只计一次）；
   - 长专注次数（同一已确认项目的连续片段达到 45 分钟）；
   - 优先成果完成（完成数 / 有优先成果计划的日期数）。
2. 每日使用当天 Bootstrap 数据；每周使用当前自然周；每月使用当前自然月。
3. 小型可信度说明显示采集天数、复盘天数和有成果项目关联的天数，不新增巨大指标卡。
4. 指标为事实和相关性，不生成生产力总分，也不因 AFK 自动判定低效。

### R3. 首页信息密度

1. 工作活动模块总高度在常用窗口中保持紧凑，目标不超过约 260px。
2. 今天注意力分布、项目/应用明细和今天时间轴继续保留，不因新增模块丢失。
3. 顶部摘要、成果证据和活动模块重新排布，常用窗口打开后应同时看到工作活动图和今天分布的入口，不出现多个同等级巨型标题。
4. 沿用组件级模块和主题 Token；重要结构变化必须在真实安装版中肉眼可见，不能只改变量名。

### R4. 更小、更安静的悬浮胶囊

1. 折叠态从当前 316×68 进一步缩小，目标约 252×48 逻辑像素；展开态目标约 268×150。
2. 折叠态只保留状态点、当前项目、连续专注时间和展开控制；移除重复的状态/计时标签。
3. 展开态保留今日该项目、实际电脑投入、占比、打开和隐藏；采用单行/紧凑布局，减少卡片嵌套。
4. 边框、光效和按钮降低存在感；状态异常仍必须可辨识。
5. 保持默认置顶、拖动、位置记忆、桌面模式、隐藏/托盘恢复和不抢焦点行为。
6. 窗口缩放后必须重新夹紧到当前显示器工作区，不能被挤到屏幕外。

### R5. 发布与升级

1. 版本升级为 v0.7.0。
2. 先合并并验证 v0.6.2 之后尚未安装的视觉基础提交，不重复实现、不回退其测试。
3. 覆盖安装后保留历史记录、计划、复盘、规则、别名、隐私规则、悬浮位置和自启动设置。
4. 更新 PRD 台账、版本规划、发布说明和使用说明。

## Acceptance criteria

- [ ] ActivityWatch 范围统计使用官方 Query API 一次提交多个时间段，不以 366 次 `getSummary` 暴力加载首页。
- [ ] 每日、每周、每月均由同一组每日事实聚合，总量守恒，固定时长分档有单元测试。
- [ ] 查询失败显示不可用状态；有记录的 0 秒与不可用状态可区分。
- [ ] 成果注意力对同一天重复关联的项目只计算一次；应用与待分类时间不被猜入成果。
- [ ] 长专注只统计达到 45 分钟的已确认项目片段。
- [ ] 首页模块支持鼠标、键盘、减少动效偏好和常用窗口宽度。
- [ ] 点击有效每日格可以打开对应日期的活动明细。
- [ ] 折叠悬浮窗不大于 260×52，展开态不大于 280×160，且窗口位置在双显示器变化后仍合法。
- [ ] 悬浮窗显示/展开不抢走当前工作窗口焦点。
- [ ] `npm test`、`npm run typecheck`、`npm run build` 通过；新增行为测试在移除实现时会失败。
- [ ] 连续两轮 Electron E2E 通过，覆盖首页活动图、范围切换、每日格下钻和悬浮胶囊。
- [ ] 安装包 smoke 通过；真实覆盖安装显示 v0.7.0，真实数据与自启动设置保留。
- [ ] Git 工作树干净，Trellis 归档，v0.7.0 tag 与 main 推送至公开仓库。

## Non-goals

- 不新增“统计”或“规律”页面承载本版核心图表。
- 不把未分类应用自动判断成工作或非工作。
- 不采集 Token、键盘正文、录屏、手机或线下活动。
- 不生成生产力评分、连续打卡惩罚、团队排行榜或员工监控视图。
- 不引入云同步、付费 API、代码签名证书或 macOS 发布。

## Stop conditions

- 需要证书、费用、真实密钥或新的账号授权。
- 发生不可逆真实数据风险、隐私采集边界扩大或远程 Git 冲突。
- 三条独立修复路径仍无法通过同一发布门禁。
