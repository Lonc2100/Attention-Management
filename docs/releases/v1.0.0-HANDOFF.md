# v1.0.0 发布门禁 Handoff

当前仓库已发布到 `v0.6.0` 本地 Windows beta。v1.0.0 不可继续自动发布，原因不是构建失败，而是两个不可由本机代码替代的发布条件未满足：

1. 代码签名：当前安装包经 Windows Authenticode 检测为 `NotSigned`。Electron Builder 可以构建未签名 installer，但这不等于安全、可信的正式公开发布；签名需要证书或受授权签名服务与账号。
2. 真实外部验证：新 profile E2E 和本机升级只能证明实现链路。它们不能证明其他 Windows 用户能理解窗口标题采集、隐私排除、备份恢复和未签名风险。

## 恢复条件

- 产品所有者提供已授权的签名方案，明确证书/服务的使用范围和发布账号权限。
- 至少一名独立 Windows 测试者完成端到端试用，并留下可复核反馈；任何严重问题进入复现—修复—回归闭环。
