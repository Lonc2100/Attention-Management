# v0.4.2 悬浮预览性能优化

## Goal

让首页圆环、图例和时间轴的悬浮预览即时、稳定，不再因为鼠标移动而出现明显卡顿。

## Requirements

- 鼠标进入目标时显示提示卡，提示卡固定在目标附近，不随目标内部的每一次鼠标移动追踪。
- 圆环、图例与时间轴继续联动高亮同一分类。
- 非当前分类可弱化，但不得再对大量时间块同时使用 CSS `filter`。
- 提示卡不得使用大面积 `backdrop-filter`；保持当前信息、层级和深色视觉。
- 键盘聚焦行为与鼠标行为一致，`prefers-reduced-motion` 仍受支持。
- 不改变 ActivityWatch、项目归属、存储、时间计算与悬浮专注窗。
- 回滚只需恢复本次 renderer/CSS diff，不涉及数据迁移。

## Acceptance Criteria

- [x] 同一目标内移动鼠标不会持续更新 React tooltip 状态。
- [x] 进入另一个目标时，提示内容与位置正确更新。
- [x] 圆环、图例和时间轴仍至少有两个同类元素进入联动激活状态。
- [x] 提示框不超出窗口边界，时间轴窄块仍能悬浮命中。
- [x] 大量非活动时间块只改变透明度，不使用 `filter`。
- [x] 单测、类型检查、构建和 Electron E2E 连续两轮通过。
- [x] v0.4.2 安装后真实用户数据与 Windows 自启动保留。

## Notes

- 方案 A 的取舍：牺牲“提示卡追随光标”，换取更低重绘成本和更稳定阅读位置。
