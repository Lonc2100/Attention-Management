# v0.7.4 · 低交互时间与统一统计口径

发布日期：2026-07-19

## 修正

- 不再把 ActivityWatch 默认的短暂无键鼠输入直接等同于离开电脑。
- 默认 15 分钟内作为“低交互推定工作”计入；达到阈值才作为主要离开排除。
- 首页、活动明细、全年工作活动、周期指标、个人规律和 Codex 项目统一使用同一口径。
- 首页和活动明细明确披露低交互推定时长与当前阈值，避免把推定伪装为确定事实。

## 可控与可逆

- 设置中可选择 5、10、15、20 或 30 分钟的主要离开阈值。
- 活动明细中的主要离开时段可人工改为“仍在工作”，也可恢复为离开电脑。
- 存储 schema 从 v7 升级到 v8，保存阈值和人工覆盖；ActivityWatch 原始事件不会被修改。
- 阈值或人工覆盖改变后，全年派生缓存按 policy key 安全重建。

## 真实数据对账

- 2026-07-19 09:42–15:59 的窗口覆盖为 22,460 秒。
- 旧口径计入 16,028 秒；15 分钟口径计入 20,166 秒，恢复 4,138 秒低交互时间。
- 两段达到阈值的主要离开合计 2,293 秒，仍保持排除，未把整段电脑开启时间冒充工作。

## 边界
