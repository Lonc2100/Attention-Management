# v0.4.0 活动明细与纠错

## 一句话目标

让“系统说我把时间花在哪里”变成一份可检查、可纠正、会逐渐学会的本地注意力账本。

## 默认交互

- 左侧新增「活动明细」，与今日概览、早间计划、晚间复盘严格分开。
- 中间上方是实际记录区间时间线，下方是按时间排列的活动。
- 点击时间块或活动行，右侧打开同一条记录的证据：时间、应用、窗口标题、归类来源与置信说明。
- 纠错时选择项目；需要时勾选“以后同类活动自动归类”。
- 规则只管理未来活动，历史记录由本次人工纠错负责，不会暗中重算。

## 六个布局方向

```text
A 三栏调查工作台：时间线 + 明细 + 证据抽屉（采用）
B 高密度表格：信息多，但日常阅读压力大
C 日历轨道：时间感强，但横向空间浪费
D 项目树：适合看总量，不适合找错分
E 只看待纠错队列：效率高，但丢失完整一天
F 上午/下午/晚上章节：叙事舒服，精确定位较弱
```

## 归类优先级

```text
人工纠错 > 第一条命中的启用规则 > 已确认 Codex 项目 > 应用/待分类
```

## 状态范围

正常、加载中、当天无记录、筛选无结果、ActivityWatch 错误、AFK 数据缺失的部分可用、超长窗口标题、大量活动、规则冲突、保存成功与失败。

## 明确不做

不改写 ActivityWatch 原始数据；不默认回溯历史；不在本版加入正则表达式、AI 自动判断、云同步和 macOS 采集。
