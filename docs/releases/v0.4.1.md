# v0.4.1 — 当前打开 Codex 聊天归属修正

v0.4.1 修复一个 P0 语义偏差：项目注意力不再跟随“最近发送过消息的聊天”，而是跟随 Codex 桌面端当前真正打开的项目聊天。

## 变化

- 点开另一个项目聊天后，约一个采样周期内自动切换项目，无需先发送消息。
- Windows UI Automation 只读取顶部项目名和聊天标题；Codex app-server 继续提供稳定线程 ID、cwd 和项目身份。
- 当前界面不可读、标题找不到或同名无法唯一消歧时，时间进入“待分类”，不再错误累计到旧项目。
- ActivityWatch 的前台、AFK 和新鲜度门控保持不变；后台 agent 运行仍不计为用户项目注意力。
- 本地数据格式保持 v4，升级不会迁移或清空既有记录、纠错与规则。

## 隐私

不录屏、不采集键盘输入、不读取聊天正文，也不保存完整 Windows 无障碍树。界面信号只在内存中用于当前标题映射。
